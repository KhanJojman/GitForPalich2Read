# Шпаргалка по статусам файлов в Git

Git отслеживает состояние файлов в вашем репозитории и присваивает им различные статусы, чтобы вы могли контролировать изменения. Вот основная информация о статусах файлов в Git:

## Основные статусы файлов:

- **Untracked (неслеживаемый)**: Файл не был добавлен в Git и не отслеживается.
- **Unmodified (неизмененный)**: Файл не был изменен с момента последнего коммита.
- **Modified (измененный)**: Файл был изменен после последнего коммита.
- **Staged (подготовленный)**: Файл был добавлен в индекс Git для будущего коммита.

## Использование:

- **Просмотр статуса файлов**: `git status` покажет текущий статус всех файлов в рабочем каталоге и индексе Git.
- **Добавление файла в индекс**: `git add <file>` добавит файл в индекс Git, переводя его статус в "Staged".
- **Удаление файла из индекса**: `git reset <file>` уберет файл из индекса Git, переводя его статус в "Modified".
- **Отмена изменений в файле**: `git checkout -- <file>` отменит все изменения в файле, возвращая его к последней сохраненной версии.
- **Игнорирование файлов**: Создайте файл `.gitignore` и добавьте в него шаблоны файлов или папок, которые должны быть проигнорированы Git.

## Примеры статусов файлов:

- `git status` - покажет статус всех файлов в рабочем каталоге и индексе Git.
- `git add file.txt` - добавит файл "file.txt" в индекс Git, переводя его статус в "Staged".
- `git reset file.txt` - уберет файл "file.txt" из индекса Git, переводя его статус в "Modified".
- `git checkout -- file.txt` - отменит все изменения в файле "file.txt", возвращая его к последней сохраненной версии.

## Дополнительные возможности:

- **Статус ветки**: Команда `git status` также покажет статус текущей ветки, указывая наличие новых коммитов или изменений.
- **Коммит изменений**: Используйте `git commit` для создания коммита из подготовленных файлов в индексе.

Это основная информация о статусах файлов в Git. Использование команды `git status` поможет вам отслеживать изменения в файлах, добавлять их в индекс и контролировать состояние вашего репозитория.
